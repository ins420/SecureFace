# INN 픽셀 수준 동작 원리
## SecureFace-RX / PRO-Face S 기술 설명

---

## 전제: INN은 픽셀값이 아닌 웨이블릿 계수로 작동

먼저 이미지를 DWT(하르 웨이블릿 변환)로 분해합니다.

**2×2 픽셀 패치 예시:**

```
원본 x:       [[200, 180],      블러 y:       [[177, 177],
              [160, 170]]                    [177, 177]]
```

DWT를 적용하면:

```
DWT(x):                          DWT(y):
 LL = (200+180+160+170)/2 = 355   LL = 354  ← 거의 같음 (저주파, 평균)
 LH = (200+180-160-170)/2 =  25   LH =   0  ← 블러하면 엣지 사라짐
 HL = (200-180+160-170)/2 =   5   HL =   0
 HH = (200-180-160+170)/2 =  15   HH =   0
```

**블러의 핵심: 고주파 계수(LH, HL, HH)를 0으로 지운다.**  
**원본 복원 = 이 고주파 계수들을 되살린다.**

---

## INN이 하는 일 (Forward, 보호)

INN은 입력을 두 덩어리로 나눕니다:

```
x1 = DWT(원본) 전체  = [LL=355, LH=25, HL=5, HH=15]  ← 원본의 4개 계수
x2 = DWT(블러) 전체  = [LL=354, LH= 0, HL= 0, HH= 0] ← 블러의 4개 계수
```

둘 다 LL, LH, HL, HH를 모두 가집니다. 차이는 **어느 이미지에서 온 계수냐**입니다.

**Affine Coupling 연산:**

```
s, t = 서브넷(x2, K)           ← 블러 계수 + 키로 변환값 계산

네트워크가 학습으로 찾아낸 s, t (각 계수별):
  LL: exp(s_LL)×355 + t_LL ≈ 354  →  t_LL ≈ -1
  LH: exp(s_LH)× 25 + t_LH ≈   0  →  t_LH ≈ -25
  HL: exp(s_HL)×  5 + t_HL ≈   0  →  t_HL ≈  -5
  HH: exp(s_HH)× 15 + t_HH ≈   0  →  t_HH ≈ -15

y1 = exp(s) × x1 + t ≈ [354, 0, 0, 0] = DWT(블러)
```

`y1 ≈ DWT(블러)` → IWT 후 ŷ ≈ 177 (블러처럼 보임)  
저장되는 값: `ŷ = 177.002` (블러 y=177과 0.002 차이)

---

## 복원 (Inverse)

저장된 `y1 ≈ [354, 0, 0, 0]`과 키 `K`로 역산:

```
x1_복원 = (y1 - t) / exp(s)
        = ([354, 0, 0, 0] - [-1, -25, -5, -15]) / exp(≈0)
        = [355, 25, 5, 15]
        = DWT(원본)  ← 완전 복원!
```

IWT 적용하면 원본 픽셀 [[200, 180], [160, 170]] 복원.

---

## 틀린 키로 복원하면?

```
K_wrong 사용 시:
  s_wrong = 0.8,  t_wrong = 100   (완전히 다른 값)

x1_wrong = (0.45 - 100) / exp(0.8)
         = -99.55 / 2.23
         = -44.6   ← 쓰레기 (원본 = 25)
```

---

## 전체 흐름 요약

```
[보호]
원본 x (픽셀 200)
    → DWT → x1=25 (고주파, 디테일)
                  + x2=354 (저주파, 블러 내용)
    → Affine: y1 = exp(s) × x1 + t    (s, t는 키K로 결정)
    → y1 ≈ 0 (블러 고주파와 유사)
    → IWT → ŷ = 177.002 (PSF에 저장)

[복원]
ŷ = 177.002 (PSF에서 로드)
    → DWT → y1 = 0.45
    → Affine 역산: x1 = (y1 - t) / exp(s)    ← 키K로 s, t 재계산
    → x1 = 25
    → IWT → x = 200 (원본 복원!)

[오복원 — 틀린 키]
ŷ = 177.002 (동일)
    → DWT → y1 = 0.45
    → Affine 역산: x1 = (y1 - t_wrong) / exp(s_wrong)
    → x1 = -44.6 (쓰레기)
    → IWT → 노이즈 출력
```

## 핵심 정리

| 항목 | 설명 |
|------|------|
| 저장되는 것 | ŷ (보호본) — 블러처럼 보이지만 픽셀값이 0.002 다름 |
| 원본 저장 여부 | **없음** — 원본은 어디에도 저장되지 않음 |
| 복원 정보의 위치 | ŷ 픽셀값 자체 (수식의 분자) |
| 키 K의 역할 | 수식의 분모 exp(s)와 이동량 t를 결정 |
| 키 없이 복원 시 | s, t가 완전히 달라져 엉뚱한 값 산출 |

> **한 줄 요약:**  
> ŷ의 픽셀값이 수식 `x = (ŷ - t) / exp(s)` 의 분자이고,  
> 키 K가 분모(exp(s))와 이동량(t)을 결정한다.  
> 키 없이는 수식 자체가 성립하지 않는다.
